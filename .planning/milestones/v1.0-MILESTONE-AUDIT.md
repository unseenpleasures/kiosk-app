## Cross-Phase Integration Issues

### 1. Cache Name Mismatch (HIGH) — PWA-03, SYNC-04

**Files:** `src/sync.js` line 14, `sw.js` line 5

`sync.js` stores product images in `caches.open('kiosk-v3')`. The service worker's `activate` handler deletes every cache key that doesn't match `CACHE_NAME` (currently `kiosk-v8`). On the next SW version bump, all product images will be silently destroyed.

**Fix:** Either rename `SYNC_CACHE_NAME` in `sync.js` to match `CACHE_NAME` in `sw.js`, or add `SYNC_CACHE_NAME` to the SW activate exclusion list.

### 2. First-Sync UX Break (HIGH) — PWA-04, CAT-01

**Files:** `src/app.js` line 237, `src/admin.js` sync handler

When the app starts with no catalogue data, `boot()` skips `initCatalogue()` and shows the sync-required screen. After successfully syncing via the admin panel, navigating to `#/` calls `renderCatalogue` but `_products` is still empty (never loaded). The catalogue appears blank until a full page reload.

**Fix:** After `syncAll` completes successfully in the admin panel, call `initCatalogue()` to reload the product array.

### 3. NEW Badge Logic Bug (MEDIUM) — CAT-06

**Files:** `src/catalogue.js` lines 72-74, `src/sync.js` line 155

`isNewCard()` compares `product.createdAt` (Shopify creation date) against `_lastSyncAt` (time the most recent sync completed). Since `lastSyncAt` is written AFTER syncing those same products, all products will always have `createdAt` older than `lastSyncAt`. The badge effectively never appears.

**Fix:** Store the previous `lastSyncAt` value before overwriting, and compare against that.

### 4. Missing CSS Class (LOW) — ANALYTICS-04

**Files:** `src/admin.js` line 542, `styles/main.css`

`admin.js` assigns `className = 'admin-stat'` to the email count paragraph, but no `.admin-stat` rule exists in `main.css`. The element renders with inherited defaults — functional but unstyled.
