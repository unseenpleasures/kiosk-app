**Create src/router.js** -- hash-based navigation dispatcher. Use `function` keyword and `var` throughout (ES2017 safe, no arrow functions).

Route table:
```javascript
var ROUTES = {
  '#/':      renderCatalogueStub,
  '#/email': renderEmailStub,
  '#/admin': renderAdminStub
};
```

`handleRoute()` function:
1. Read `var hash = window.location.hash || '#/'`
2. Check exact match in ROUTES first -- if found, call the handler and return
3. Check prefix `hash.indexOf('#/category/') === 0` -- call `renderCategoryStub(hash.replace('#/category/', ''))`
4. Check prefix `hash.indexOf('#/card/') === 0` -- call `renderCardStub(hash.replace('#/card/', ''))`
5. Fallback: set `window.location.hash = '#/'` (redirects unknown hashes to home)

`initRouter()` function:
1. `window.addEventListener('hashchange', handleRoute)`
2. Call `handleRoute()` synchronously to dispatch on initial load (Pitfall 4 from RESEARCH.md)

**Screen stub render functions** (Phase 3-5 will replace these with real implementations):
Each stub function clears `#app` innerHTML and creates a simple div showing the screen name. Pattern:
```javascript
function renderCatalogueStub() {
  var app = document.getElementById('app');
  app.innerHTML = '';
  var screen = document.createElement('div');
  screen.className = 'screen';
  screen.id = 'screen-catalogue';
  var heading = document.createElement('h1');
  heading.className = 'stub-heading';
  heading.textContent = 'Catalogue';
  screen.appendChild(heading);
  app.appendChild(screen);
}
```

Create similar stubs for: `renderEmailStub` (id: screen-email, text: "Email Sign Up"), `renderAdminStub` (id: screen-admin, text: "Admin Panel"), `renderCategoryStub(categoryId)` (id: screen-category, text: "Category: " + categoryId), `renderCardStub(cardId)` (id: screen-card, text: "Card: " + cardId).

Each stub screen div gets `className = 'screen'` and a unique `id` attribute for testability.

**Create src/idle.js** -- inactivity timer with countdown overlay. Use `function` keyword and `var` throughout.

Module-level state:
```javascript
var _idleTimer = null;
var _countdownTimer = null;
var _countdownEl = null;
var _countdownRemaining = 10;
var _active = false;
```

`resetIdleTimer()`:
- If `!_active`, return immediately
- If `_countdownEl` exists, call `cancelCountdown()`
- `clearTimeout(_idleTimer)`
- `var timeout = Config.getIdleTimeout() * 1000`
- `_idleTimer = setTimeout(startCountdown, timeout)`

`startCountdown()`:
- Set `_countdownRemaining = 10`
- Call `_countdownEl = showCountdownOverlay(_countdownRemaining)`
- Start `_countdownTimer = setInterval(...)` that decrements `_countdownRemaining`, calls `updateCountdownOverlay(_countdownEl, _countdownRemaining)`, and when `<= 0` calls `cancelCountdown()` then sets `window.location.hash = '#/'`

`cancelCountdown()`:
- `clearInterval(_countdownTimer)`, set `_countdownTimer = null`
- Remove `_countdownEl` from DOM if it exists and has parentNode
- Set `_countdownEl = null`

`showCountdownOverlay(seconds)`:
- Create overlay div with `className = 'idle-overlay'`
- Create paragraph with `className = 'idle-message'`, text: `'Returning to home in ' + seconds + ' seconds...'`
- Create cancel button with `className = 'idle-cancel'`, text: `'Cancel'`, click handler calls `cancelCountdown()` then `resetIdleTimer()`
- Append to `document.body` (not #app -- overlay must be above everything including chrome)
- Return the overlay element

`updateCountdownOverlay(el, seconds)`:
- Find `.idle-message` inside el, update textContent to `'Returning to home in ' + seconds + ' seconds...'`
- Handle singular: if seconds === 1, use `'second'` not `'seconds'`

`initIdleTimer()`:
- Set `_active = true`
- Add event listeners on `document`: `touchstart`, `touchend`, `pointerdown` -- all with `{ passive: true }` -- calling `resetIdleTimer`
- Call `resetIdleTimer()` to start the initial timer

`pauseIdleTimer()`:
- Set `_active = false`, `clearTimeout(_idleTimer)`, cancel any active countdown

`resumeIdleTimer()`:
- Set `_active = true`, call `resetIdleTimer()`
