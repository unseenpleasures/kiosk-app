Replace the Phase 1 stub `src/config.js` with the full localStorage wrapper module. Use `function` keyword throughout (no arrow functions). Use `var` declarations.

**Key constants:**
```javascript
var KEYS = {
  PASSCODE_HASH: 'kiosk_passcode_hash',
  EVENT_NAME:    'kiosk_event_name',
  EVENT_DATE:    'kiosk_event_date',
  IDLE_TIMEOUT:  'kiosk_idle_timeout'
};

var DEFAULT_IDLE_TIMEOUT = 60;
```

**Config object with typed getter/setter pairs:**
- `Config.getPasscodeHash()` — returns `localStorage.getItem(KEYS.PASSCODE_HASH)` or `null`
- `Config.setPasscodeHash(hash)` — `localStorage.setItem(KEYS.PASSCODE_HASH, hash)`
- `Config.getEventName()` — returns stored string or `''`
- `Config.setEventName(name)` — stores string
- `Config.getEventDate()` — returns stored string or `''`
- `Config.setEventDate(date)` — stores string
- `Config.getIdleTimeout()` — returns stored integer or `DEFAULT_IDLE_TIMEOUT` (60). Parse with `parseInt(stored, 10)`.
- `Config.setIdleTimeout(seconds)` — stores `String(seconds)`

**SHA-256 passcode hashing (two standalone functions, not on Config object):**

`hashPasscode(passcode)` — returns `Promise<string>` (hex string):
1. `var encoder = new TextEncoder();`
2. `var data = encoder.encode(passcode);`
3. `return window.crypto.subtle.digest('SHA-256', data).then(function(hashBuffer) { ... })`
4. Convert ArrayBuffer to hex: `Array.from(new Uint8Array(hashBuffer)).map(function(b) { return b.toString(16).padStart(2, '0'); }).join('')`
5. Do NOT use `Uint8Array.prototype.toHex()` — not available in Safari 16.x

`verifyPasscode(inputPasscode)` — returns `Promise<boolean>`:
1. Call `hashPasscode(inputPasscode)`
2. Compare result to `Config.getPasscodeHash()`
3. Return `storedHash !== null && inputHash === storedHash`

**Critical notes:**
- `crypto.subtle` requires secure context (HTTPS or localhost) — this is satisfied by the PWA serving pattern
- No default passcode is set. `getPasscodeHash()` returns null on fresh install. Phase 3 admin panel handles the "no passcode" first-use flow.
- All localStorage access in the entire app MUST go through this module (STATE.md constraint)
