**Function: `renderEmail()`** — main entry point called by router for `#/email`.

**Function: `renderEmailForm(card)`** — builds form elements programmatically inside the card.
Elements in order:
1. `h2.email-screen-heading` — textContent: `"Sign Up for Updates"`
2. `p.email-screen-subheading` — textContent: `"Get early access to new designs and exclusive event discounts."`
3. `input.email-input` — type `"email"`, inputmode `"email"`, autocorrect `"off"`, autocapitalize `"off"`, spellcheck `"false"`, placeholder `"your@email.com"`, id `"email-input"`

   - `input.email-consent-checkbox` type `"checkbox"` (do NOT set `checked`), id `"email-consent"`
   - `span` with textContent: `"I agree to receive email updates from The ID Card Factory."` (exact GDPR wording per D-05)

Wire checkbox `change` event: set `submitBtn.disabled = !checkbox.checked`

Wire submit button `click` event: call `validateAndSave(emailInput, checkbox, card)`

**Function: `validateAndSave(emailInput, checkbox, card)`**
- Trim the email value; validate: `email.length > 0 && email.indexOf('@') !== -1` (per D-07)
- If invalid: show error element with textContent `"Please enter a valid email address."`, `style.display = 'block'`, return
- Hide error element
- Build record: `{ email: email, eventName: Config.getEventName(), eventDate: Config.getEventDate(), consentAt: new Date().toISOString() }` (per D-12)
- Call `dbAdd('emails', record)` — on `.then()` call `showEmailConfirmation(card)`
- On `.catch(err)`: log `console.error('Email save failed:', err, record)`, show error message `"Something went wrong saving your details. Please try again."` in `--color-destructive` below submit button, re-enable submit button, do NOT clear form fields

**Function: `showEmailConfirmation(card)`**
- Clear card contents: `card.innerHTML = ''` (safe — no user content in the card at this point)
- Build confirmation elements in order:
  1. `span.email-confirmation-icon` — textContent `"\u2713"` (checkmark), display block
  2. `h2.email-confirmation-heading` — textContent `"YOU'RE IN!"` (uppercase per UI-SPEC — Bebas Neue)
  3. `p.email-confirmation-body` — textContent `"Thanks for signing up. We\u2019ll let you know about new designs and event exclusives."`
  4. `p.email-confirmation-count` — id `"email-count"`, `style.display = 'none'` — populated async below
  5. `p.email-countdown` — id `"email-countdown"`, textContent `"Returning to catalogue in 5\u2026"`
  6. `button.btn-secondary.btn-large` — textContent `"Back to browsing"`, type `"button"`

- **Async signup count:** Call `dbGetAll('emails').then(function(all) { ... })` — filter by `r.eventName === Config.getEventName()`, set count element textContent to `"You're #" + filtered.length + " to sign up at this event!"`, set `style.display = 'block'`. On `.catch()` silently ignore (nice-to-have).

- **Countdown:** Set `var remaining = 5`. Use `setInterval` at 1000ms, stored in `_emailCountdown`. Each tick: decrement `remaining`, update countdown textContent to `"Returning to catalogue in " + remaining + "\u2026"`. When `remaining <= 0`: `clearInterval(_emailCountdown)`, `_emailCountdown = null`, `window.location.hash = '#/'`.

- **Early-dismiss button click:** `clearInterval(_emailCountdown)`, `_emailCountdown = null`, `window.location.hash = '#/'`

- **Stale interval cleanup (CRITICAL per RESEARCH.md Pitfall 1):** Add a one-time `hashchange` listener immediately after starting the countdown:
  ```javascript
  function cleanupCountdown() {
    if (_emailCountdown !== null) {
      clearInterval(_emailCountdown);
      _emailCountdown = null;
    }
    window.removeEventListener('hashchange', cleanupCountdown);
  }
  window.addEventListener('hashchange', cleanupCountdown);
  ```
