# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: PWA Foundation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Phase 1 delivers two screens only: the branded splash screen and the
> "Sync required" blocking screen. No catalogue, no admin panel, no nav
> chrome. All tokens defined here are project-wide — every subsequent
> phase inherits them.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between label and icon |
| sm | 8px | Compact element spacing, label internal padding |
| md | 16px | Default element spacing, paragraph margin |
| lg | 24px | Section padding, content block vertical rhythm |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Major section breaks, splash vertical centering offset |
| 3xl | 64px | Page-level spacing, splash top/bottom inset on landscape viewport |

Exceptions:
- Touch targets: minimum 48px in every dimension for all interactive elements (KIOSK_APP.md §UI/UX Design Direction). This overrides the scale where needed — a button may use 48px height even if no scale token maps to exactly 48.
- QR code placement inset: 24px from corner edges (lg) — consistent across Phase 1 and Phase 2 global chrome.

Source: KIOSK_APP.md §UI/UX Design Direction (48px minimum), standard 8-point scale applied

## Typography

| Role | Family | Size | Weight | Line Height | Usage |
|------|--------|------|--------|-------------|-------|
| Display | Bebas Neue | 48px | 400 (Bebas Neue is a single-weight display font) | 1.1 | Splash app name "ID Card Factory" |
| Heading | Inter | 28px | 700 | 1.2 | Screen headings (e.g. "Sync Required" heading) |
| Body | Inter | 16px | 400 | 1.5 | Descriptive copy, blocking screen explanation text |
| Label | Inter | 14px | 700 | 1.2 | Buttons, CTAs, UI chrome labels |

Notes:
- Bebas Neue is a display-only face with a single weight; treat its effective weight as bold (400 maps to visually bold output). Do not use it at sizes below 32px.
- Inter 400 and Inter 700 are the only two weights loaded. No other weights are permitted — do not add 500 or 600 (adds woff2 download weight with no design benefit for this project).
- All font files are self-hosted woff2. No CDN calls, no fallback to Google Fonts network.
- `font-display: swap` is mandatory on all `@font-face` declarations — ensures text visible during font load on A9X.

Source: RESEARCH.md §Fonts §Pattern 7, CONTEXT.md §Specific Ideas (bold display font requirement)

## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-bg` | `#0d0f1a` | Primary background — fills 100% of every screen, manifest `background_color`, splash screen fill, `apple-touch-startup-image` background |
| Secondary (30%) | `--color-surface` | `#0e1b4d` | Cards, modal surfaces, panel backgrounds (Phase 2+). In Phase 1: not rendered — token defined for forward compatibility |
| Accent (10%) | `--color-accent` | `#f5c842` | Reserved for: (1) primary CTA button background on splash screen, (2) "Sync required" heading text, (3) active/pressed state ring on interactive elements |
| Text primary | `--color-text-primary` | `#f0f0f0` | All body text, labels, descriptive copy |
| Text secondary | `--color-text-secondary` | `#a0a8c0` | Sub-labels, secondary descriptive text, hint copy |
| Destructive | `--color-destructive` | `#e53e3e` | Destructive confirmation actions only (not used in Phase 1 — token defined for forward compatibility) |

Accent reserved for (explicit list):
1. Primary CTA button background on splash screen ("Browse our collection →")
2. "Sync Required" heading text on the blocking screen
3. Interactive element pressed/active outline (2px ring, Phase 2+)

Accent is NOT used for: decorative borders, secondary buttons, background gradients, icon fills, or general link colours.

## Screen Specifications

### Screen 1: Branded Splash Screen

Triggered when: IndexedDB health check returns `true` (catalogue data present). In Phase 1 this screen is always shown after the health check completes, as the boot sequence calls `showSplashScreen()` when data is present.

Layout (landscape viewport 1366 × 1024 CSS px):
- Full-bleed background: `--color-bg` (`#0d0f1a`)
- Logo: `assets/logo.svg`, centred horizontally, top 50% minus offset (visually upper-centre), max-width 320px
- App name: "ID Card Factory" in Bebas Neue 48px `--color-text-primary`, below logo with 24px (lg) gap
- Primary CTA: "Browse our collection →" — button, Inter 14px weight 700, `--color-bg` text on `--color-accent` background, 48px height minimum, 24px horizontal padding, centered horizontally, 32px (xl) below app name
- QR code: `assets/qr-code.png`, fixed to bottom-right corner, 24px (lg) inset from both edges, 96px × 96px rendered size. Label above in Inter 12px `--color-text-secondary`: "Visit our website"

Interaction:
- Tapping anywhere on the splash screen (not just the CTA button) navigates to `#/` catalogue root — per D-10. In Phase 1 this is a navigation stub with no catalogue view behind it.
- CTA button: no hover state (touch device). Pressed state: `--color-accent` at 80% opacity (CSS `opacity: 0.8` on `:active`).

Entry animation:
- Splash screen fades in from transparent to fully opaque over 300ms using CSS `opacity` transition (`ease-in`). Defined in `styles/animations.css`.
- No JS animation. No transforms. Opacity transition only — safest for A9X GPU.

Source: CONTEXT.md D-08, D-09, D-10; KIOSK_APP.md §UI/UX Design Direction

---

### Screen 2: "Sync Required" Blocking Screen

Triggered when: IndexedDB health check returns `false` (no catalogue data — fresh install or post-eviction). Per PWA-04 success criterion.

Layout (landscape viewport 1366 × 1024 CSS px):
- Full-bleed background: `--color-bg` (`#0d0f1a`)
- Logo: `assets/logo.svg`, centred horizontally, max-width 240px, positioned upper-centre
- Heading: "Sync Required" — Inter 28px weight 700, `--color-accent` (`#f5c842`), below logo with 24px (lg) gap
- Body copy: "Catalogue data not found. Please sync before the event." — Inter 16px weight 400, `--color-text-primary`, 1.5 line-height, centered, max-width 480px, 16px (md) below heading
- No sync button — admin panel not built yet (D-11). No interactive elements on this screen.
- No QR code on this screen — QR code is part of global chrome (Phase 2); this blocking screen is pre-chrome.

Entry animation:
- No animation on this screen. Renders immediately on mount — operator must see the blocking state without delay. Static render, no opacity transition.

Source: CONTEXT.md D-11; REQUIREMENTS.md PWA-04; ROADMAP.md Phase 1 Success Criterion 3

## Interaction Constraints (A9X / Safari 16 Specific)

These constraints apply to all phases and are established here as the project-wide interaction baseline.

| Constraint | Rule | Rationale |
|------------|------|-----------|
| Hover states | No `hover` pseudo-class for functional interactions | Touch device — no cursor. Use `:active` for press feedback only. |
| Animations | CSS `transition` and `opacity`/`transform` only. No JS animation, no `@keyframes` with `transform3d` complexity. | A9X GPU ceiling; keep composited properties minimal |
| Touch targets | 48px minimum in both dimensions | KIOSK_APP.md explicit requirement |
| Focus styles | Visible `:focus-visible` outline using `--color-accent` 2px offset | Basic contrast requirement (REQUIREMENTS.md out-of-scope note still requires basic contrast) |
| `will-change` | Do not use `will-change` speculatively — only apply to actively animating elements | A9X memory pressure |
| `position: fixed` | Maximum 2 fixed elements per screen | Safari on iPadOS 16 has known jank with many fixed elements during scroll |
| `backdrop-filter` | Do not use | Not supported on A9X / Safari 16 at acceptable performance |

Source: CLAUDE.md §Safari/iPadOS-Specific Gotchas §Performance (A9X Chip), KIOSK_APP.md §UI/UX Design Direction
