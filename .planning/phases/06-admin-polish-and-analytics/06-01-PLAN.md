Add two new functions to src/admin.js:

1. **aggregateAnalytics(records, currentEventName)** — Pure data function. Takes the full analytics array and current event name. Returns an object:
   ```javascript
   {
     topCards: [{ id, name, count }],      // top 10 by view count, descending
     topSearches: [{ query, count }],      // all unique queries, descending by count
     zeroResults: [string]                  // unique query strings that returned zero results
   }
   ```
   Implementation:
   - Filter records where `r.eventName === currentEventName`
   - For topCards: iterate filtered records where `r.type === 'card_view'`, count by `r.cardId`, track `r.cardName` in a separate lookup object, sort descending by count, slice(0, 10)
   - For topSearches: iterate filtered records where `r.type === 'search'`, count by `r.query`, sort descending by count
   - For zeroResults: iterate filtered records where `r.type === 'search' && r.zeroResult === true`, collect unique query strings into an object then `Object.keys()`
   - Use `var` declarations, `function` keyword callbacks, no arrow functions (ES2017 constraint)

2. **renderAnalyticsSummarySection(panel, exitBtn)** — DOM-building function. Creates the "Event Analytics" admin section and inserts it before exitBtn.
   - Create section div with className `admin-section`
   - Create h2 with className `admin-section-heading`, textContent `'Event Analytics'`
   - Show current event name scope: create a paragraph with className `sync-result-detail` showing `'Showing data for: ' + eventName` (or `'No event configured'` if eventName is empty)
   - Create a loading paragraph with textContent `'Loading analytics...'`
   - Call `Promise.all([dbGetAll('analytics'), dbGetAll('emails')])` then in the `.then()`:
     - Remove the loading paragraph (`loading.style.display = 'none'`)
     - Calculate `emailCount` by filtering emails where `r.eventName === eventName` and taking `.length`
     - Call `aggregateAnalytics(analyticsRecords, eventName)`
     - Build DOM sub-sections:
       a. **Total Emails:** paragraph with className `admin-stat`, textContent `'Emails captured: ' + emailCount`
       b. **Top 10 Most-Viewed Cards:** h3 with textContent `'Top 10 Most-Viewed Cards'`, then an ordered list (`ol`) element. For each item in `topCards`, create an `li` with textContent `item.name + ' — ' + item.count + ' views'`. If topCards is empty, show a paragraph `'No card views recorded.'`
       c. **Most-Searched Terms:** h3 with textContent `'Most-Searched Terms'`, then an ordered list. For each item in `topSearches`, create an `li` with textContent `item.query + ' — ' + item.count + ' searches'`. If topSearches is empty, show a paragraph `'No searches recorded.'`
       d. **Zero-Result Searches:** h3 with textContent `'Zero-Result Searches'`, then an unordered list (`ul`). For each query in `zeroResults`, create an `li` with textContent being the query string. If zeroResults is empty, show a paragraph `'No zero-result searches.'`
   - Use `panel.insertBefore(section, exitBtn)` to place the section before the exit button

3. **Wire into renderAdminPanel:** In the `renderAdminPanel` function, after the `renderEmailExportSection(panel)` call (line 312) and after the exitBtn is created and BEFORE `panel.appendChild(exitBtn)` (line 326), add:

   - Then call `renderAnalyticsSummarySection(panel, exitBtn)` which uses `insertBefore`

   This follows the same pattern established by Phase 5's email export section insertion.
